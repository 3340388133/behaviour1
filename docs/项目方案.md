# 可疑人员识别系统 - 项目方案

## 一、系统架构

```
视频输入
    ↓
┌─────────────────────────────────────────────────────────┐
│  检测层：人脸/人头检测                                    │
│  - RetinaFace（精度优先）                                │
│  - YOLOv8-face / YOLOv8-Head（实时性优先）               │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│  跟踪层：多目标跟踪（带 Re-ID）                           │
│  - StrongSORT / DeepOCSORT / BoT-SORT                   │
│  - 关键：遮挡后能重新识别同一人                           │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│  姿态估计层：头部姿态估计                                 │
│  - HopeNet（常规角度）                                   │
│  - WHENet（大角度 |Yaw|>60° 时）                         │
│  - 输出：Yaw, Pitch, Roll 三轴角度                       │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│  识别层：可疑行为识别  ★★★ 创新点在这里 ★★★              │
│  - 输入：姿态序列 + 外观特征 + 运动特征                   │
│  - 输出：正常 / 可疑（张望）/ 异常                        │
└─────────────────────────────────────────────────────────┘
```

## 二、可疑行为判定规则（基线）

| 条件 | 判定 |
|------|------|
| \|Yaw\| > 30° 且 \|Pitch\| < 20° | **张望（可疑）** |
| \|Yaw\| > 30° 且 \|Pitch\| > 30° | 异常低头/抬头（弱告警） |
| 其他 | 正常 |

## 三、识别层创新点方案

### 创新点 1：时序 Transformer 行为建模

**动机**：
- 基线方法用简单阈值规则，无法捕捉时序模式
- 可疑张望通常有特定的时序特征（反复、快速、不自然）

**方法**：
```
姿态序列 [T, 3] → Positional Encoding → Transformer Encoder → 行为分类
         ↑
    (Yaw, Pitch, Roll) × T帧
```

**技术细节**：
- 输入：连续 T 帧的姿态角度 (如 T=32, 约1秒)
- 编码：1D Positional Encoding + Linear Projection
- 主干：2-4 层 Transformer Encoder
- 输出：[CLS] token → MLP → 二分类

**预期优势**：
- 能学习"反复左右看"的时序模式
- 自注意力机制自动关注关键帧
- 比 LSTM 更好的长程依赖建模

---

### 创新点 2：Coordinate Attention 替代 CBAM

**动机**（来自需求文档）：
- CBAM 更适合语义区域强化，对几何回归任务效果不明显
- 头部姿态估计是几何任务，需要方向感知更强的注意力

**方法**：
```python
# Coordinate Attention
class CoordAtt(nn.Module):
    def __init__(self, inp, oup, reduction=32):
        # 分别在 H 和 W 方向做 pooling
        self.pool_h = nn.AdaptiveAvgPool2d((None, 1))
        self.pool_w = nn.AdaptiveAvgPool2d((1, None))
        # 融合后生成方向感知的注意力权重
        ...
```

**应用位置**：
- 姿态估计网络的 backbone 中
- 或识别层的特征融合模块中

---

### 创新点 3：多模态时序融合

**动机**：
- 单纯姿态角度可能不够鲁棒
- 结合外观变化和运动速度可提高判别力

**方法**：
```
┌─────────────┐   ┌─────────────┐   ┌─────────────┐
│  姿态分支    │   │  外观分支    │   │  运动分支    │
│ (Yaw,P,R)   │   │ (Re-ID feat)│   │ (光流/速度) │
└──────┬──────┘   └──────┬──────┘   └──────┬──────┘
       │                 │                 │
       └────────────┬────┴────────────────┘
                    ↓
            Cross-Modal Attention
                    ↓
              行为分类器
```

**特征说明**：
- 姿态分支：[T, 3] 角度序列
- 外观分支：[T, 512] Re-ID 特征（来自跟踪器）
- 运动分支：[T, 2] 头部移动速度/加速度

---

### 创新点 4：对比学习预训练

**动机**：
- 标注数据有限（视频级标注）
- 利用大量无标注视频进行自监督学习

**方法**：
```
同一人不同时刻的姿态序列 → 正样本对
不同人的姿态序列 → 负样本对

Loss = InfoNCE(anchor, positive, negatives)
```

**实现**：
1. 预训练阶段：对比学习学习姿态序列表示
2. 微调阶段：冻结 encoder，训练分类头

---

### 创新点 5：不确定性感知的多任务学习

**动机**：
- 需要同时预测：分类结果 + 姿态回归 + 置信度
- 不同任务的损失量纲不同，需要自适应平衡

**方法**：
```python
# 可学习的不确定性权重
log_sigma_cls = nn.Parameter(torch.zeros(1))
log_sigma_reg = nn.Parameter(torch.zeros(1))

loss = (1 / (2 * sigma_cls**2)) * L_cls + \
       (1 / (2 * sigma_reg**2)) * L_reg + \
       log(sigma_cls) + log(sigma_reg)
```

## 四、实验验证方案

### 消融实验设计

| 实验 | 方法 | 对照 |
|------|------|------|
| A1 | Transformer 时序建模 | 阈值规则 |
| A2 | Transformer | LSTM |
| A3 | Coordinate Attention | CBAM / 无注意力 |
| A4 | 多模态融合 | 单姿态输入 |
| A5 | 对比学习预训练 | 随机初始化 |

### 评价指标

- **准确率 (Accuracy)**
- **召回率 (Recall)**：可疑行为的检出率
- **精确率 (Precision)**：报警的准确性
- **F1-Score**
- **AUC-ROC**

## 五、数据集要求（来自需求文档）

1. 覆盖不同头部角度区间
2. 包含自然动作，不是刻意表演
3. 标注到视频级即可（不必逐帧）
4. 正常样本占多数（符合真实分布）
5. 站立 60% + 行走 40%

## 六、推荐实施顺序

1. **第一阶段**：用 BoT-SORT 完成数据集构建
2. **第二阶段**：实现基线（阈值规则）
3. **第三阶段**：实现创新点 1（时序 Transformer）
4. **第四阶段**：添加创新点 2-5 作为消融实验
5. **第五阶段**：整理论文
